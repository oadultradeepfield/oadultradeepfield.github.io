## Week 1: Compiler, Types, Classes, Objects

- Java code can be executed either by being interpreted by the Java Interpreter or compiled into bytecode, which is then interpreted by the JVM. It cannot be interpreted directly by the JVM.
- Compilers can only analyze code without running it. This means they cannot detect whether a particular statement will be executed. Run-time errors occur while a program is running, after it has been compiled, so they cannot be detected by compilers.
- A reference variable needs to be initialized to an object, such as `Circle c1 = new Circle()`. Simply doing `Circle c1` assigns a special reference value `null`. The `new Circle()` instantiates a new object in memory and points `c1` to it. This means that doing `Circle c2 = c1` will reference `c2` to the same object.
- $T <: S$ if and only if the code written for $S$ can safely be used for $T$ without any loss of information. Subtyping is also a partial order. The intuition we need refers to the piece of code rather than the data. For example, a piece of code written for 32-bit data cannot handle 64-bit data due to overflow, but the opposite is true. That is why 32-bit data are considered subtypes of 64-bit data, rather than the opposite direction.
- The subtyping order is: `boolean` $<:$ `byte` $<:$ `short` $<:$ `int` $<:$ `long` $<:$ `float` $<:$ `double`, and `char` $<:$ `int`.
- In this example:

  ```java
  float f = 2.5;
  int i = 5;
  f = i;
  ```

  `f` is assigned the value `5` instead of being assigned the type `i`, as Java is strongly and statically typed. This is called widening type conversion since we are converting `5` (an `int`) to a `float` (subtype to supertype). The reverse process is called narrowing (supertype to subtype).

## Week 2: Class Instances/Methods, Inheritance

- A constructor cannot have a return type, so specifying `void` will result in an error.
- `static` is the keyword used to describe class variables (not tied to instances). Therefore, static methods cannot access non-static variables.
- In `public static void main(String[] args)`, the **method signature** is `public static void main(String[] args)`, and the **method descriptor** is `public static void`.
- Overriding is the concept of having the same method signature in both parent and child classes, while overloading involves having the same method name with different parameters within the same class. Adding `@Override` helps detect potential mistakes at compile time.
